# MBG: Graph-Enhanced Mamba foundation model for EEG

MBG introduces graph-aware spatial processing into the EEG foundation model paradigm established by CBraMod and EEGMamba. The architecture stacks GEM (Graph-Enhanced Mamba) blocks, each combining multi-head graph attention across EEG channels with per-channel Mamba temporal scanning, fused via a learned gate. The PatchEmbedding and output interface match the CBraMod/EEGMamba signatures for downstream task compatibility. A CPU fallback replaces Mamba's SSM with a pointwise MLP when `mamba-ssm` is unavailable.

**Watch for:** The bidirectional flip in MixerModel with `n_layer_mamba=1` (odd) reverses the temporal dimension, causing the fusion gate to blend spatially-correct and temporally-reversed representations — a confirmed architectural bug. The `model_for_tuab.py` hardcodes `num_channels=22` in the backbone but `16*10*200` in the classifier, guaranteeing a crash when the TUAB dataset provides 16 channels. PatchEmbedding's Conv2d arithmetic only produces `d_model` features when `in_dim=200` exactly, making the `in_dim` parameter misleading.

## High-level view

The GEM block's temporal branch outputs a reversed sequence when using a single Mamba layer (`n_layer_mamba=1`), misaligning it with the spatial branch at the fusion point. EEGMamba avoids this by using `n_layer=12` (even) where flips cancel; each GEM block's internal `n_layer_mamba=1` does not cancel.

The Mamba fallback replaces the selective state-space mechanism with `Linear → GELU → Linear`, preserving shape compatibility but losing all temporal modeling capacity. On CPU the model degenerates to graph attention + two feedforward paths with a learned mixing gate.

The task-specific head (`model_for_tuab`) introduces a channel dimension mismatch: the backbone graph modules are sized for 22 channels while the classifier's flattened linear layers assume 16 channels and 10 time segments — a guaranteed runtime crash on TUAB data.

PatchEmbedding's Conv2d kernel/stride arithmetic is hardwired to `in_dim=200`. The `in_dim` constructor parameter suggests configurability that doesn't exist.

## Temporal reversal through MixerModel's bidirectional flip

The `MixerModel.forward` loop flips `hidden_states` and `residual` along the sequence dimension after each layer iteration. With `n_layer=12` (even), the flips cancel out and the output maintains input ordering. Each GEM block creates its own `MixerModel` with `n_layer_mamba=1`, meaning a single flip persists into the final output.

In the GEM block forward pass, `z_spatial` preserves the original temporal ordering (graph attention operates per-timestep independently), while `z_temporal` exits the Mamba path with positions reversed. The fusion gate then computes:

```python
gate = sigmoid(W_g * [z_spatial, z_temporal])
fused = gate * z_spatial + (1 - gate) * z_temporal
```

At position `t`, this blends spatial information about timestep `t` with temporal information about timestep `T-1-t`. Verified by perturbing specific input positions and observing which output positions change: zeroing `x[0, 2]` in a 3-step sequence changes `y[0, 0]` but not `y[0, 2]`, confirming the reversal.

The fix is either setting `n_layer_mamba=2` (adding parameters) or inserting `z_temporal = z_temporal.flip(2)` after the rearrange back to `(B, C, T, D)` in `gem_block.py` line ~100.

## num_channels mismatch between backbone and task head

`model_for_tuab.py` creates the MBG backbone with the default `num_channels=22`, which sizes `A_prior` to `(22, 22)`. The classifier linear layers assume `16 * 10 * 200 = 32000` input features, meaning the actual TUAB data has 16 channels.

When 16-channel input arrives, `x_graph` will be `(B*T, 16, D)` while `A_prior` is `(22, 22)`. The blending operation `alpha * self.A_prior.unsqueeze(0) + (1 - alpha) * A_learned` will fail because `A_prior` (22×22) cannot broadcast against `A_learned` (B*T, 16, 16).

## PatchEmbedding Conv2d output width

The temporal path uses `Conv2d(1, 25, kernel=(1,49), stride=(1,25), padding=(0,24))`:

```
out_w = floor((in_dim + 48 - 49) / 25) + 1
```

| in_dim | out_w | 25 × out_w | d_model match |
|--------|-------|------------|---------------|
| 100    | 4     | 100        | No            |
| 200    | 8     | 200        | Yes           |
| 400    | 16    | 400        | No            |

The subsequent `.view(bz, ch_num, patch_num, self.d_model)` assumes exactly `d_model` features. This is the same constraint as EEGMamba, but MBG's `__init__` signature accepts `in_dim` as a free parameter, implying flexibility that doesn't exist.

## Soft adjacency masking — no hard topology

The prior adjacency uses Gaussian kernel weights with `sigma=1.0` over normalized `[-1, 1]` electrode coordinates. Adjacent electrodes (distance ~0.2-0.4) get weights ~0.92-0.7; distant ones (distance ~2.0) get ~0.14. With the GAT's `1e-6` masking threshold, effectively all electrodes remain connected — the adjacency acts as a soft weighting rather than a hard topology constraint. If sparse graph structure is intended for regularization or efficiency, `sigma` should be reduced or a top-k sparsification added.

## Fallback mixer and model degeneracy on CPU

`SimpleMambaFallback` operates independently at each position — no cross-timestep interaction. The "temporal" branch of the GEM block contributes nothing beyond a per-position nonlinear transform, identical in kind to what the FFN already provides. The model degenerates to graph attention + two feedforward paths with a learned gate controlling their mixing ratio.

The bidirectional flip still physically reverses positions due to residual connections, meaning the temporal reversal bug (issue #1) also affects the fallback path.

## sys.path manipulation at module level

Both `gem_block.py` and `graph_construction.py` insert `sys.path.insert(0, '.')` at the top. This works when run from the project root but breaks when imported from another directory or when installed as a package. A root-level `__init__.py` with proper package structure would eliminate this fragility.
